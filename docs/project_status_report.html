LivingBetterCC — Reporte de estado del proyecto

Django 5.1.x CKEditor5 FormBuilder
Fecha: 2025-12-27

1) Resumen ejecutivo

Aplicación Django con múltiples apps (authentication, classroom, dashboard, shop, etc.) y un sistema de formularios dinámicos (authentication.formbuilder).

  • El proyecto corre sin errores en manage.py check.
  • Migración completa de CKEditor4 (django-ckeditor) a CKEditor5 (django_ckeditor_5).
  • Corrección de NoReverseMatch del upload endpoint de CKEditor5.
  • Corrección de configuración para evitar URLConf equivocado.
  • Fix de subida de imágenes en FormBuilder (uso de request.FILES + multipart/form-data).
  • Soporte de dos imágenes laterales diferentes (image_left/image_right) en el encabezado del formulario.
  • Detalle de formulario completado alineado visualmente con el render + botón “Imprimir”.

2) Objetivos logrados (cambios principales)

2.1 CKEditor: migración total a CKEditor5

Problemas iniciales:

  • NoReverseMatch por nombre de URL de upload mal configurado.
  • Warning de seguridad de CKEditor4.

Acciones:

  • Ajuste de CK_EDITOR_5_UPLOAD_FILE_VIEW_NAME al nombre real de la vista de upload de django_ckeditor_5.
  • Eliminación de django-ckeditor y migración de campos rich text a CKEditor5Field.
  • Limpieza de migraciones antiguas para remover imports de ckeditor.

2.2 Bootstrap vs CKEditor5 (toolbar no aparecía)

Causa: varios formularios sobre-escribían widget.attrs['class'] = 'form-control' y borraban la clase django_ckeditor_5 requerida para inicializar el editor.

Solución: patrón de “append class” (agregar sin borrar) y fix de __init__ mal indentados en formularios de cursos.

2.3 URLConf incorrecto (404 usando otro proyecto)

Solución: se forzó DJANGO_SETTINGS_MODULE='livingbettercc.settings' en manage.py.

2.4 Subida de imágenes en FormBuilder

Causa: el form del CRUD no tenía enctype="multipart/form-data" y las vistas no pasaban request.FILES al ModelForm.

Solución: se corrigieron plantilla y vistas para soportar upload real.

2.5 Dos imágenes laterales distintas (izquierda/derecha)

  • Se agregaron image_left y image_right a FormDefinition.
  • Se mantuvo image como legacy (compatibilidad).
  • Plantillas ajustadas para mostrar izquierda/derecha.
  • Migración aplicada: formbuilder.0009_...image_left...image_right.

2.6 Detalle de formulario completado (UI + imprimir)

  • Header reestructurado para coincidir con el layout centrado del render.
  • Botón “Imprimir” con window.print().

3) Estado actual por módulo

  • CRUD de definiciones de formulario (FormDefinition) funcional.

classroom / dashboard / shop

  • Campos rich text migrados a CKEditor5.
  • Forms ajustados para no romper clases del widget.
